# Техническое задание на проведение аналитического исследования и разработку прогнозной модели рынка жилой недвижимости г. Москвы

## 1. Общие положения и цели проекта
Целью данного проекта является выявление ключевых ценообразующих факторов на рынке недвижимости Москвы и разработка математической модели для оценки рыночной стоимости объектов. 
**Бизнес-задачи:**
* Оптимизация процесса оценки объектов для внутреннего использования.
* Повышение прозрачности ценообразования для конечных клиентов через внедрение инструментов автоматизированного расчета.
* Определение степени влияния инфраструктурных факторов (близость к метро) и физических характеристик здания на ликвидность объекта.

## 2. Описание используемых данных
Анализ проводится на основе предоставленного датасета, содержащего следующие атрибуты объектов:
* **Стоимость:** Цена в рублях (`price`).
* **Локация:** Административный округ (`region_of_moscow`), время в пути до метро пешком в минутах (`min_to_metro`).
* **Физические характеристики:** Общая площадь (`total_area`), жилая площадь (`living_area`), количество комнат (`number_of_rooms`), высота потолков (`ceiling_height`).
* **Параметры здания:** Год постройки (`construction_year`), этажность дома (`number_of_floors`), этаж расположения объекта (`floor`).
* **Статусы:** Тип жилья (новостройка/вторичное — `is_new`), юридический статус (квартира/апартаменты — `is_apartments`).

## 3. Требования к аналитической части
В ходе исследования необходимо дать количественно обоснованные ответы на следующие вопросы:
1.  **Факторный анализ:** Определить веса каждого параметра в формировании итоговой цены. Требуется интерпретация влияния в денежном или процентном эквиваленте.
2.  **Анализ транспортной доступности:** Выявить характер зависимости цены от времени в пути до метро. Определить, сохраняется ли «премия за близость к метро» в разных административных округах в равной степени.
3.  **Влияние юридического статуса:** Провести сравнительный анализ стоимости объектов для стандартных квартир и апартаментов с учетом характеристик дома и локации.

## 4. Требования к разработке модели
* **Интерпретируемость:** Модель не должна функционировать по принципу «черного ящика». Алгоритм должен позволять восстановить логику формирования цены для каждого предсказания.
* **Обработка аномалий:** В рамках подготовки данных необходимо предусмотреть алгоритмы выявления и устранения выбросов (некорректные площади, ошибки ввода в годе постройки, экстремальные значения цен).
* **Метрики качества:** Точность модели должна быть подтверждена статистическими метриками (например, $R^2$, MAE). Необходимо определить границы применимости модели и среднюю величину погрешности.

## 5. Формат представления результатов
По итогам работы должны быть представлены:
1.  **Аналитический отчет:** Документ, включающий визуализацию ключевых зависимостей и выводы по бизнес-вопросам.
2.  **Программный код:** Реплицируемый скрипт или Jupyter Notebook на языке Python с комментариями к этапам обработки данных и обучения модели.
3.  **Демонстрационный прототип:** Интерактивный интерфейс, обеспечивающий ввод параметров объекта и получение прогнозной стоимости в режиме реального времени.

## 6. Критерии приемки
* Наличие аргументированного обоснования выбора финального алгоритма.
* Визуальное подтверждение качества работы модели на тестовой выборке.
* Корректная обработка пропущенных значений и аномалий в исходных данных.
